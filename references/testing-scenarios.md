# 测试场景参考

## 端到端测试矩阵

本技能建议在以下 5 个核心场景上进行端到端测试，覆盖咨询报告、创业融资、技术演讲、政策研究、产品发布五个差异极大的场景。

---

## 场景 1：AI 行业研究报告

| 属性 | 值 |
|------|-----|
| 页数 | 15 页 |
| 场景 | 咨询/董事会/战略 |
| 品牌主题偏好 | consulting, financial |
| 视觉主角 | 数据可视化型为主 |

**必须覆盖的图表类型（7 种）：**
- [ ] 瀑布图（利润拆解）
- [ ] 堆叠柱状图（收入构成）
- [ ] 雷达图（厂商能力对比）
- [ ] 漏斗图（产业链环节）
- [ ] 折线图（增长率趋势）
- [ ] 环形图（市场份额）
- [ ] 面积图（市场规模累积）

**测试检查清单：**
- [ ] 数据页是否清晰展示 KPI 关键指标
- [ ] 图表配色是否符合 consulting 主题
- [ ] 信息密度是否合理（多为高密度）
- [ ] 数据来源是否有标注
- [ ] 封面是否体现"咨询报告"风格

## 场景 2：AI 创业 BP

| 属性 | 值 |
|------|-----|
| 页数 | 12 页 |
| 场景 | 路演/融资/A 轮 |
| 品牌主题偏好 | pitch-vc, product-launch |

**必须覆盖的图表类型（5 种）：**
- [ ] 柱状图（市场规模）
- [ ] 折线图（增长预测）
- [ ] 漏斗图（用户转化）
- [ ] 环形图（收入结构）
- [ ] 堆叠柱状图（成本分解）

**测试检查清单：**
- [ ] 灵魂句是否在第一页清晰表达
- [ ] 叙事弧线是否有"问题→方案→验证→展望"结构
- [ ] 路演风格是否足够"抓人"
- [ ] 对比页是否用 TPL-COMPARISON 模板
- [ ] 结尾是否有"行动呼吁"页面

## 场景 3：AI 技术发布会

| 属性 | 值 |
|------|-----|
| 页数 | 10 页 |
| 场景 | 技术演讲/GPU/算力 |
| 品牌主题偏好 | tech-launch |

**必须覆盖的图表类型（5 种）：**
- [ ] 柱状图（性能对比）
- [ ] 雷达图（多维度能力展示）
- [ ] 面积图（算力增长趋势）
- [ ] 散点图（性能-功耗关系）
- [ ] 仪表盘（技术指标达成率）

**测试检查清单：**
- [ ] 封面是否突出技术感（深色背景/亮色强调）
- [ ] 仪表盘是否用刻度/阈值标注
- [ ] 散点图是否有趋势线/标注
- [ ] TPL-HERO 是否用于关键性能指标强调
- [ ] 整体风格是否为"技术演讲"风格

## 质量验收标准

| 等级 | 标准 | 行动 |

| ✅ 通过 | 所有 26 项 TC 通过 + 5 场景全部跑通 | 交付 |
